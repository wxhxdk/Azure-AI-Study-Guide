# 第 9 章 Text and Speech AI

> 本章覆盖自然语言处理（NLP）与语音 AI 的典型任务，并训练从输入和预期输出判断能力类型。

---

# 1. 学习目标

学习后，你应能区分文本分析、文本生成、语音识别、语音合成与语音翻译，并识别相应的业务场景。

# 2. Natural Language Processing

NLP 用于理解和分析已有文本。常见任务包括：

| 任务 | 输出示例 |
| --- | --- |
| 情绪分析 | 正面、负面、中性 |
| 关键短语提取 | “退款延迟”“物流问题” |
| 命名实体识别 | 人名、地点、日期、组织 |
| 语言检测 | 中文、日文、英文 |
| 文本分类 | 投诉、咨询、订单问题 |

```text
客户评论 → 情绪分析 → 负面
新闻文本 → 实体识别 → 人名、地点、日期
服务工单 → 文本分类 → 技术支持
```

# 3. Speech AI

Speech AI 的输入或输出是声音：

* **Speech-to-Text**：将语音转换为文字，例如会议转写与实时字幕；
* **Text-to-Speech**：将文字合成为语音，例如无障碍阅读和语音导航；
* **Speech Translation**：把一种语言的语音转换成另一种语言的文本或语音。

```text
音频 → Speech-to-Text → 文本
文本 → Text-to-Speech → 音频
```

# 4. 文本分析与生成式 AI

文本分析重点是理解已有内容；生成式 AI 重点是创建新内容。

```text
“这条评论是正面还是负面？” → NLP 情绪分析
“请根据评论写一封回复”     → Generative AI
```

# 5. 实际场景

* 呼叫中心：转写通话、分析情绪、为客服生成摘要；
* 无障碍服务：把页面文本朗读出来；
* 舆情分析：批量识别评论主题和情绪；
* 多语言会议：将说话内容转写并翻译。

# 6. Responsible AI 注意事项

语音和语言系统应测试不同语言、口音、表达方式和环境噪声下的表现。涉及录音、身份或敏感对话时，应进行告知、最小化收集、访问控制与安全存储。
