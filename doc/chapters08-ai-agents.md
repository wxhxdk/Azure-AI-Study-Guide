# 第 8 章 AI Agents

> AI Agent（AI 智能体）是在模型能力基础上，能根据指令、上下文和工具完成目标导向任务的应用模式。

---

# 1. 学习目标

学习后，你应能说明 AI Agent 的组成、工作流程、工具的作用，以及它与普通聊天机器人的区别。

# 2. 什么是 AI Agent

AI Agent 不只生成一段回答，还能在受控边界内决定下一步：检索信息、调用工具、使用结果并继续完成任务。

```text
用户目标
   ↓
Agent Instructions + Model
   ↓
选择合适的 Tool / 数据源
   ↓
执行并读取结果
   ↓
生成答复或完成动作
```

# 3. 核心组成

## 3.1 Instructions

Instructions 定义 Agent 的角色、目标、行为边界、工具使用规则以及失败时的处理方式。例如：“只能查询当前用户有权限查看的订单；无法确认时转人工。”

## 3.2 Model

模型负责理解任务、决定何时需要工具，并将工具结果组织成自然语言或结构化输出。

## 3.3 Tools

工具使 Agent 能访问外部能力，例如搜索知识库、查询订单、创建工单、计算金额或调用 API。工具必须具有明确输入、输出和访问控制。

## 3.4 Knowledge / Context

知识库和上下文为 Agent 提供任务所需事实。与其依赖模型记忆，更可靠的做法是从经过授权、更新的数据源中检索信息。

# 4. Agent 与 Chatbot 的区别

| 普通 Chatbot | AI Agent |
| --- | --- |
| 主要围绕对话回复 | 围绕目标完成多步骤任务 |
| 通常不调用外部系统 | 可以在许可范围内调用工具 |
| 侧重问答 | 可检索、判断、执行和汇报 |

并非每个聊天应用都需要 Agent。若任务仅是简单 FAQ，受控问答应用通常更简单、成本更低且更易治理。

# 5. 工具调用的安全原则

* 最小权限：只能访问完成任务必需的数据和动作；
* 明确边界：禁止高风险或不可逆操作自动执行；
* 参数验证：不要让自然语言直接变成未验证的系统操作；
* 人工确认：付款、删除、权限变更等操作应由人确认；
* 记录审计：保留工具调用、结果和责任链。

# 6. 场景示例

**订单助手**：用户询问物流状态，Agent 识别订单号，调用订单查询工具，再根据真实结果回答。  
**IT 支持助手**：先检索知识库；若无法解决，则创建工单并附上已收集的诊断信息。  
**旅行规划助手**：可整合检索与日程工具，但下单前需要用户最终确认。

# 7. 测试与评估

测试 Agent 时除了评价回答质量，还要检查：工具是否被正确选择、参数是否安全、权限是否得到遵守、失败时是否能够停止或升级人工，以及多步骤链路是否可追踪。

# 8. 练习题

## 练习题 1

应用需要按当前用户权限查询订单系统，然后回复物流状态。最关键的 Agent 组件是：

A. Tool 与访问控制  
B. OCR  
C. 图像分类  
D. 聚类

<details><summary>查看答案</summary>

**答案：A。** Agent 需要受控工具来查询真实业务数据。

</details>

## 练习题 2

哪项最能区分 Agent 与只进行文本生成的聊天机器人？

A. 可以使用受控工具完成多步骤任务  
B. 必须使用图像输入  
C. 不需要模型  
D. 永远不需要人工监督

<details><summary>查看答案</summary>

**答案：A。** Agent 的关键特征是目标导向的编排与工具使用。

</details>

# 9. 一句话记忆

> **AI Agent = 模型 + 指令 + 知识/上下文 + 受控工具；它可以为完成目标执行多步骤工作，但权限、人工确认和审计不能省略。**
