# 第 11 章 Information Extraction

> 信息提取将发票、表单、合同和扫描文档中的内容转为业务可用的结构化数据。

---

# 1. 学习目标

学习后，你应能解释信息提取与 OCR 的关系，识别文档处理场景，并理解为什么结果需要验证和人工复核。

# 2. 什么是信息提取

信息提取从半结构化或非结构化内容中找出具有业务意义的字段、关系和布局，并输出结构化结果。

```text
输入：发票 PDF
    ↓
文字、表格、键值对、布局识别
    ↓
输出：供应商、发票日期、税额、总金额、明细行
```

# 3. OCR 与信息提取

| 能力 | 关注点 |
| --- | --- |
| OCR | 将图片中的字符读成文本 |
| 信息提取 | 识别字段含义、布局、表格和关系 |

```text
“总计：¥1,280”被读出             → OCR
¥1,280 被标记为 invoice_total 字段 → Information Extraction
```

# 4. 常见场景

* 发票和收据自动录入；
* 从合同中提取当事方、日期和条款；
* 从申请表中读取姓名、地址和编号；
* 归档扫描文档并建立可检索字段；
* 从大量业务资料中抽取结构化信息供后续流程使用。

# 5. 质量与人工复核

文档质量、语言、手写内容、复杂表格和模板变化都可能影响结果。应保存置信度和原始来源，对低置信度字段、金额、身份信息等高影响字段设置人工审核。

```text
文档输入 → 提取字段 → 置信度检查
                      ├─ 高置信度 → 业务流程
                      └─ 低置信度 → 人工复核
```

# 6. 隐私与安全

文档常含个人信息、财务或合同数据。应最小化收集，按角色授予访问权限，保护存储和传输，并明确保留和删除规则。

# 8. 一句话记忆

> **OCR 负责读出文字；信息提取负责理解文档中的字段、表格和业务含义，并将其转为结构化数据。**
