# 0.1 为什么写这本手册

近年来，以大语言模型（Large Language Model，LLM）为代表的生成式人工智能（Generative AI）迅速发展，正在改变软件开发的方式。从智能问答、代码生成，到企业知识检索、文档处理和智能代理（Agent），AI 已逐渐成为现代软件开发的重要组成部分。

Microsoft Azure 提供了完整的 AI 平台和丰富的云服务，使开发者能够快速构建企业级 AI 应用，而无需自行训练大型模型。从 Azure OpenAI 到 Azure AI Search，再到 Vision、Speech、Document Intelligence 等服务，Azure AI 已形成一套覆盖多种业务场景的完整生态。

然而，对于刚开始学习 Azure AI 的开发者来说，最大的困难往往不是不会编程，而是不知道应该如何开始。

学习过程中常常会遇到这样的疑问：

- Azure AI 到底包含哪些服务？
- Azure OpenAI、Azure AI Search、Azure AI Vision 之间有什么关系？
- AI-900 与 AI-102 分别适合什么人？
- Microsoft Learn 应该如何配合学习？
- 学完认证之后，又该如何真正应用到项目开发中？

官方文档内容丰富且权威，但知识点分散，更偏向产品说明和功能介绍。对于初学者来说，很难快速建立完整的知识体系。

因此，本手册希望以开发者的视角，对 Azure AI 的知识进行系统整理，帮助读者建立完整的学习框架，从理解概念开始，逐步过渡到实践开发，再进一步完成认证和项目应用。

本手册不是微软官方文档的替代品，而是希望成为连接官方文档与实际开发之间的一座桥梁。

# 0.2 本书适合哪些读者

本手册主要面向以下读者：

- 希望系统学习 Azure AI 的开发者；
- 准备 Azure AI 相关认证（AI-900、AI-102 或后续版本）的学习者；
- 希望从传统软件开发转向 AI 应用开发的工程师；
- 希望了解 Azure AI 服务体系和应用场景的技术人员。

如果你已经具备 C++、Java、C#、Python 等任意一种编程语言的基础，阅读本手册会更加轻松。

如果没有 Azure 使用经验，也没有关系。本手册会从最基础的 Azure 概念开始介绍，帮助你逐步建立完整的知识体系。

# 0.3 如何使用本手册

建议按照以下学习顺序进行：

1. 阅读章节内容，理解核心概念。
2. 配合 Microsoft Learn 完成官方学习模块。
3. 根据实验章节完成 Azure Portal 实践。
4. 阅读示例代码，学习 SDK 的基本使用方法。
5. 完成本章练习题，检查学习效果。
6. 在实际项目中尝试应用所学知识。

建议不要跳过基础章节。

Azure AI 建立在 Azure 云平台之上，理解 Azure 的基本概念，将有助于后续学习 Azure AI 服务和开发实践。

# 0.4 本书的特点

本手册具有以下特点：

## 官方资料优先

所有知识点均以 Microsoft Learn 和微软官方文档为主要参考来源，确保内容准确、可靠，并及时跟进 Azure AI 的更新。

## 开发者视角

不仅介绍服务的使用方式，更关注其设计思想、适用场景以及实际开发中的应用。

## 图文结合

对于 Azure 架构、AI 服务关系、RAG 工作流程等抽象内容，将通过示意图、流程图和架构图帮助理解，而不仅仅依赖文字说明。

## 理论结合实践

每章都会配套实验、示例代码和练习题，使学习内容能够真正落地。

## 持续更新

Azure AI 的产品和认证体系仍在快速发展。本手册将持续维护，并根据微软官方更新不断完善内容。

# 0.5 关于 Azure AI 认证

Azure AI 认证能够帮助学习者建立明确的学习目标，也是检验学习成果的一种方式。

但是，认证并不是学习的终点。

随着 Azure AI 产品持续演进，考试内容和认证体系也可能发生变化。真正长期受益的能力，是理解 Azure AI 的基本原理、掌握 Azure 平台的资源管理方式，以及能够根据业务需求合理选择 AI 服务。

因此，本手册将认证作为学习过程中的阶段性目标，而不是最终目标。
